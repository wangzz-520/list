# 上线前测试报告

生成日期：2026-05-17

## 结论

当前项目已完成上线前核心收口：清单详情页主流程可用，模板内容已补充并与云端初始化模板同步，云开发仍保持本地缓存降级能力。项目未接入微信支付，未强制手机号授权，未引入传统服务器或前端 npm 依赖。

## 1. 清单详情页最终打磨

### 模板清单页

- 支持勾选模板事项，并显示“已选择”进度。
- 支持给生成后的清单取名。
- 支持快速添加事项，并可选择添加到指定分组。
- 支持删除模板事项或新增事项，删除前有二次确认。
- 支持重置当前清单，恢复模板初始状态并清除临时新增事项。
- 支持设为常用清单，入口已放在标题说明下方的明确操作行。
- 点击“生成我的清单”后：
  - 只保存已勾选事项和自定义新增事项；
  - 保存为本地自定义清单，并通过同步机制进入云端 `user_data.data.customLists`；
  - 清空该模板草稿，避免再次进入仍保留已生成前的勾选状态；
  - 尝试生成清单图片并保存到相册；
  - 完成后跳转回首页。

### 最近生成清单页

- 从“我的”页进入最近生成清单时，页面为只读展示。
- 不显示清单名称编辑、常用、收藏、进度、快速添加事项、删除事项和重置按钮。
- 清单事项按分组以一行多项的标签式布局展示，避免页面过长。
- 保留“分享清单”按钮，并独占一行。

### 分享清单

- 云开发可用时，分享前调用 `createShareList` 生成共享快照。
- 分享打开时优先读取 `shareId` 对应的共享清单。
- 云开发不可用时降级为普通路径分享。

## 2. 模板内容整理

### 当前模板规模

- 本地模板数量：25 个。
- 云端初始化模板数量：25 个。
- 分类数量：8 个。
- 本地 `data/templates.js` 与云端 `cloudfunctions/initTemplates/templates.js` 模板 ID 已同步。

### 本次补充模板

- `办事证件清单`：覆盖办证、入职、报销、学校提交材料等高频材料准备场景。
- `周末亲子出行清单`：覆盖公园、博物馆、短途亲子出行场景。

### 内容边界

- 老人、母婴、就医相关模板仅保留生活准备提醒，不包含诊疗建议。
- 防诈骗模板仅做风险自查提醒，不涉及金融建议。
- 未新增支付、商城、交易、派单或 AI 生成类功能。

- `project.config.json` 已配置 `cloudfunctionRoot: "cloudfunctions/"`。

  - `login`

  - `initTemplates`
  - `getTemplates`

  - `submitFeedback`

## 6. 已知风险

- 保存清单图片依赖用户相册权限；用户拒绝授权时，清单仍会保存到“我的清单中心”，但图片不会进入相册。
- 云函数需要在微信开发者工具中部署到同一云环境，否则云端模板、反馈、分享快照和用户数据同步不可用。
- `initDatabase` 创建集合能力受云开发环境权限影响；如创建失败，需要在云开发控制台手动创建对应集合后再调用 `initTemplates`。
