# AI 员工团队说明：本地缓存版

当前项目是微信小程序“万能生活清单助手”本地缓存版。

统一边界：

- 不接微信支付
- 不接云后台
- 不使用云函数
- 不使用云数据库
- 不引入传统服务器
- 不引入前端 npm 依赖
- 用户数据全部保存在本地缓存中

## 固定员工

1. `product-planner`：产品策划，负责功能边界、版本规划。
2. `miniprogram-developer`：小程序开发，负责原生微信小程序实现。
3. `ui-designer`：UI 设计，负责界面、布局、样式和体验优化。
4. `checklist-content-operator`：清单内容运营，负责扩充模板和内容风险控制。
5. `qa-tester`：测试验收，负责上线前检查和测试报告。
6. `growth-compliance`：合规增长，负责个人主体边界、分享增长和审核材料。

## 协作要求

- 不同员工不要同时大规模修改同一个文件。
- 内容运营修改 `data/templates.js` 后，测试验收需要检查模板字段完整。
- UI 设计修改 WXML/WXSS 后，小程序开发必须检查绑定变量和事件名。
- 小程序开发修改本地存储结构后，必须兼容旧数据。
- 合规增长提出不能做的功能，其他员工必须遵守。

## 常用任务

### 小程序开发

检查当前项目是否可运行，重点检查 `app.json` 页面路径、`project.config.json`、JS require 路径、WXML 绑定变量、分享路径。本阶段不要接云后台，不要引入 npm，不要接微信支付。发现问题请直接修复，并输出验证步骤。

### UI 设计

在不改变业务逻辑的前提下优化当前小程序 UI。重点优化首页、分类卡片、详情页进度区域、我的页面列表。保持主色 `#2f6fed`。

### 内容运营

扩充或精修 `data/templates.js`。新增模板必须包含 `id`、`title`、`category`、`icon`、`heat`、`description`、`groups`。

### 测试验收

输出 `docs/release_test_report.md`。重点检查页面路径、JS 语法、WXML 绑定、本地存储 key、空状态、分享路径、用户隐私风险、是否误触微信支付或登录授权。能直接修复的低风险问题请修复。

### 合规增长

准备上线审核材料、推广文案、个人主体边界说明。不要设计诱导分享、付费交易、壁纸下载外链、医疗金融法律结论类能力。
