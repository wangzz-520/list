# Codex 项目规则：万能生活清单助手

## 项目定位

这是一个微信小程序项目，名称为“万能生活清单助手”。

目标：

- 个人开发者可上线
- 不接微信支付
- 不接云后台
- 不使用云函数、云数据库、传统服务器
- 用户数据全部保存在微信小程序本地缓存中
- 用生活清单解决真实场景问题
- 优先体验、留存、分享，而不是复杂功能

## 技术规则

1. 使用原生微信小程序技术：WXML、WXSS、JavaScript、JSON。
2. 不引入传统服务器，不接微信支付。
3. 不使用微信云开发、云函数、云数据库。
4. 不强制用户授权手机号、头像、昵称。
5. 本地模板放在 `data/templates.js`。
6. 本地缓存能力放在 `utils/storage.js`。
7. 清单计算、分组、进度等通用逻辑放在 `utils/checklist.js`。
8. 页面逻辑不得直接写大量重复数据，优先复用工具函数。
9. 页面必须能在微信开发者工具中直接编译。

## 本地存储规则

1. 用户私有数据只保存在本机缓存中。
2. 不获取 openid。
3. 不上传清单、历史记录到服务器。
4. 不提供意见反馈入口，避免用户误以为反馈会提交给开发者。
5. 清理微信缓存、卸载小程序或更换设备后，本地数据可能丢失。
6. 涉及存储格式变更时，必须兼容旧数据或写清迁移逻辑。

## 产品规则

1. 所有功能围绕“防遗漏”“可勾选”“可分享”“可复用”。
2. 首页必须让用户 3 秒内知道怎么用。
3. 清单详情页必须保留：勾选、进度、添加事项、删除事项、重置、分享。
4. 新增模板时必须包含：id、title、category、icon、heat、description、groups。
5. 模板内容要偏生活工具，不要做医疗诊断、金融建议、法律结论。
6. 涉及老人、母婴、就医等场景时，只能做准备清单和生活提醒。
7. 不要做头像生成、文案生成、AI 生成器类功能。
8. 不要做接单交易、派单收费、在线支付、商城团购。

## UI 规则

1. 风格：干净、清爽、工具型、亲和。
2. 主色：蓝色 `#2f6fed`。
3. 卡片圆角、足够留白、字体清楚。
4. 重要按钮用 `primary-btn`，次要按钮用 `secondary-btn`。
5. 不要使用过度复杂的动画。
6. 页面要适配普通手机屏幕，避免横向溢出。

## 代码质量规则

1. 修改后检查 WXML 标签闭合。
2. 修改后检查 JS 是否存在未定义变量。
3. 修改后检查 `app.json` 的 pages 配置是否正确。
4. 新增页面必须同时添加 `.js`、`.wxml`、`.wxss`、`.json`。
5. 不要删除已有核心功能。
6. 不要新增云函数、云数据库、云配置文件。
