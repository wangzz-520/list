# Codex 项目规则：万能生活清单助手

## 项目定位

这是一个微信小程序项目，名称为“万能生活清单助手”。

目标：

- 个人开发者可上线
- 不接微信支付
- 已接入微信云开发 CloudBase
- 本地缓存优先，云端同步增强
- 用生活清单解决真实场景问题
- 优先体验、留存、分享，而不是复杂功能

## 技术规则

1. 使用原生微信小程序技术：WXML、WXSS、JavaScript、JSON。
2. 不引入传统服务器，不接微信支付。
3. 云后台使用微信云开发能力：云函数、云数据库、云存储预留。
4. 不强制用户授权手机号、头像、昵称。
5. 本地模板放在 `data/templates.js`。
6. 云端模板初始化逻辑放在 `cloudfunctions/initTemplates`。
7. 云函数统一放在 `cloudfunctions/`。
8. 云环境配置放在 `config/cloud.js`。
9. 本地缓存能力放在 `utils/storage.js`。
10. 云函数调用封装放在 `utils/cloudApi.js`。
11. 云端用户数据同步放在 `utils/cloudSync.js`。
12. 清单计算、分组、进度等通用逻辑放在 `utils/checklist.js`。
13. 页面逻辑不得直接写大量重复数据，优先复用工具函数。
14. 页面必须能在微信开发者工具中直接编译。

## 云开发规则

1. 必须保留本地缓存降级能力：云环境未配置或云函数失败时，核心功能仍能使用。
2. 云函数返回值统一包含 `ok` 字段。
3. 用户私有数据只能按 `_openid` 隔离读写。
4. 前端不要直接读写 `user_data`、`feedbacks`、`shared_lists`，优先走云函数。
5. `checklist_templates` 可由云函数读取；未来如需前端直读，必须先检查数据库权限。
6. 新增云函数时必须包含 `index.js` 和 `package.json`。
7. 修改云数据库字段时，必须同步更新 `docs/DB_SCHEMA.md`。
8. 修改部署流程时，必须同步更新 `docs/CLOUD_BACKEND_SETUP.md` 和 `docs/CLOUD_DEPLOY_CHECKLIST.md`。
9. 不要把真实云环境 ID、密钥、账号信息硬编码进仓库。
10. 不要删除 `config/cloud.js` 的云开发开关。

## 产品规则

1. 所有功能围绕“防遗漏”“可勾选”“可分享”“可复用”。
2. 首页必须让用户 3 秒内知道怎么用。
3. 清单详情页必须保留：勾选、进度、添加事项、删除事项、重置、分享。
4. 新增模板时必须包含：id、title、category、icon、heat、description、groups。
5. 模板内容要偏生活工具，不要做医疗诊断、金融建议、法律结论。
6. 涉及老人、母婴、就医等场景时，只能做准备清单和生活提醒。
7. 不要做头像生成、文案生成、AI 生成器类功能。
8. 不要做接单交易、派单收费、在线支付、商城团购。

## UI 规则

1. 风格：干净、清爽、工具型、亲和。
2. 主色：蓝色 `#2f6fed`。
3. 卡片圆角、足够留白、字体清楚。
4. 重要按钮用 `primary-btn`，次要按钮用 `secondary-btn`。
5. 不要使用过度复杂的动画。
6. 页面要适配普通手机屏幕，避免横向溢出。

## 代码质量规则

1. 修改后检查 WXML 标签闭合。
2. 修改后检查 JS 是否存在未定义变量。
3. 修改后检查 `app.json` 的 pages 配置是否正确。
4. 新增页面必须同时添加 `.js`、`.wxml`、`.wxss`、`.json`。
5. 不要删除已有核心功能。
6. 涉及存储格式变更时，必须兼容旧数据或写清迁移逻辑。
7. 涉及云函数修改时，必须检查云端返回结构与前端调用是否一致。

## AI 员工团队规则

本仓库已加入可在 Codex 中使用的 AI 员工配置，入口文件如下：

```text
员工总说明：docs/AI_EMPLOYEES.md
团队配置：.codex/teams/life-checklist-team.yaml
可复制任务：.codex/tasks/EMPLOYEE_PROMPTS.md
员工身份：.codex/agents/*.md
```

### 固定员工

1. `product-planner`：产品策划，负责整体方案、功能边界、版本规划。
2. `miniprogram-developer`：小程序开发，负责原生微信小程序代码实现。
3. `cloudbase-backend-developer`：微信云开发后端，负责云函数、数据库、同步、部署文档。
4. `ui-designer`：UI 设计，负责界面、布局、样式和体验优化。
5. `checklist-content-operator`：清单内容运营，负责扩充模板和内容风险控制。
6. `qa-tester`：测试验收，负责上线前检查和测试报告。
7. `growth-compliance`：合规增长，负责个人主体边界、分享增长和广告准备。

### 分工顺序

优先按以下顺序安排任务：

```text
第 1 批：产品策划 + 合规增长，先输出整体方案和边界
第 2 批：小程序开发 + 云开发后端 + 测试验收，确保本地和云端均可运行
第 3 批：清单内容运营 + UI 设计，扩内容和优化体验
第 4 批：小程序开发 + 云开发后端 + 测试验收，做功能增强和回归测试
第 5 批：合规增长 + 测试验收，做上线准备
```

### 协作要求

- 不同 AI 员工不要同时大规模修改同一个文件。
- 内容运营修改 `data/templates.js` 后，云开发后端需要同步检查 `cloudfunctions/initTemplates/templates.js`。
- UI 设计修改 WXML/WXSS 后，小程序开发必须检查绑定变量和事件名。
- 小程序开发修改本地存储结构后，云开发后端必须检查同步字段兼容。
- 云开发后端修改云函数返回结构后，小程序开发必须检查前端调用。
- 合规增长提出不能做的功能，其他员工必须遵守。
